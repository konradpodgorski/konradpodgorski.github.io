Run ‘Find in path’ within your IDE for plugin directory and look after ‘wp-content’, if you don’t find it you are probably good to go.

Find in path might have slightly different name if different IDE’s, right click on plugin directory and look for something similar.

If you find it, then you can fork plugin on github if it has repository there or simply email it’s developer that he really should fix this. Every plugin or theme developed for WordPress should use content_url() function
http://codex.wordpress.org/Function_Reference/content_url

In past you could use WP_CONTENT_URL constant but this caused a problems with websites that uses SSL certificates

]]> By: George http://konradpodgorski.com/blog/35/how-to-change-wp-content-folder-name-in-wordpress/#comment-766 George Sun, 03 Mar 2013 18:41:02 +0000 http://konradpodgorski.com/?p=35#comment-766 I don’t know if protocol has changed since the publication of this article, but using the define function did not work for me without first encapsulating the definition in parenthesis like so:

define (‘WP_CONTENT_FOLDERNAME’, ‘your-content’);
define (‘WP_CONTENT_DIR’, ABSPATH . WP_CONTENT_FOLDERNAME) ;
define (‘WP_CONTENT_URL’, ‘http://your-domain/’.WP_CONTENT_FOLDERNAME);
define (‘WP_PLUGIN_DIR’, WP_CONTENT_DIR . ‘/plugins’);
define (‘WP_PLUGIN_URL’, WP_CONTENT_URL.’/plugins’);

Afterwards it worked perfectly! Thanks for the article.
